Technical Post: Cycle, Cycle, Ad Infinitum

Alright, let's talk about Enumarable#cycle. It's written like this
array = [1, 2, 3]
array.cycle {|x| puts x}
Which would then yeild this:
1, 2, 3, 1, 2, 3, 1, 2, 3 #And it will go on forever
If it will go on forever, what's the point? It'll just run until it runs out of memory and crashes the whole system. Doesn't seem too useful. But what if we did this?
array.cycle(3) {|x| puts x}
That would yeild this:
1, 2, 3, 1, 2, 3, 1, 2, 3
It repeats the array three times, and now has a limit. So now it won't crash the system. That's a relief. So what do we do now?

A More Practical Example.

Here's the code:
some_months = ["Jan", "Feb", "March", "April"]
some_counter = some_months.cycle do |month|
puts month
break if month == "April"
end

Which will yeild:
"Jan"
"Feb"
"March"
"April"
So we can run it until it ends. What if we want to run it one at a time?
some_counter = some_months.cycle
some_counter.next Which will yeild:
"Jan"
And if we do it again?
"Feb"
Hmmm. That's pretty useful! So we can use it to create loops of code. If you want to go backwards a bit, just do:
some_counter.rewind
Which will yeild:
"Jan"
So we can go backwards and forwards. If we reach the end, it will actually wrap around and start over. But remember! If you don't put a limiter on the thing, it will run forever. Luckily, .next yeilds an element one at a time. Same with .rewind.

Other Things to Consider

Cycle will also work with two dimensional arrays, and if you use the .zip method to combine two single dimensional arrays, then use cycle to show each element (which is another array), and the grouped info. It also works with hashes, another way to show grouped data.
Cycle can work as a counter for info, or just a straight up counter.
Fool around with it! Experiment! Just remember, if you get stuck in a loop, use ctrl+c to break the run.
