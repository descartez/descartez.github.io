Technical Post: Class Warfare

So today we're gonna learn about classes and their instance variables. So what better way to learn about classes than to play a game? We have solitaire, or chess, or... Thermonuclear warfare. One of these things is not like the other. In fact one of these things is a very overt reference to the movie "Wargames", starring a very young Matthew Broderick. If you haven't seen it, you're missing out. Still has the most accurate depiction of hacking despite it being made in the golden age of the floppy disk. The big ones.

Initialize

I made a little game here, very simple. You start out with a population, and you can choose how many to put in bunkers. If you want to, you can even kick people out if they displease you. All while missiles rain down. Remember! These are your comrades! The people! You must protect them from capitalist scum!

First, let's make the class. We'll need a population of course. 

class Coldwar

attr_accessor :total_population, :underground_pop, :aboveground_pop

	def initialize(population)
		@aboveground_pop = population
		@underground_pop = 0
	end

Initialize is important. It's how we get started on this thing.It means that if we do this: test = Coldwar.new(5) That means that it will create a new Class with @total_population equal to five. That means we'll have something to work with when we introduce other methods.You'll also notice a few things here. Those things with the @ symbol. Those are instance variables. The reason why we put that little @ symbol is so that these variables can be accessed by other methods throughout the class. Handy, considering we will want to update the populations (aboveground and belowground) each time we call other methods. (Don't worry about attr_accessor, that's just a little trick that programmers use if they want to change something later. Look it up if you're curious.)

The Bunker 

def bunker(people)
	if people > @total_population
		raise ArgumentError.new("You can't bunker 
					more that you have.")
	else
		@underground_pop += people
		@aboveground_pop -= people
	end
end

def unbunker(people)
	if people > @total_population
		raise ArgumentError.new("You can't unbunker
					more that you have.")
	else
		@underground_pop -= people
		@aboveground_pop += people
	end
end

See how we made two methods here? They do pretty much the same thing. Except bunker takes the amount of people you want, adds them to your underground population and subtracts that amount from your aboveground population. Unbunker does the opposite, taking people out of your bunker and pushing them outside. In case one of your citizens snore or something. I also added an error message, so if you try to put more people underground than you have it will tell you.

The Missile Strike 

def missile
	if @aboveground_pop > 0
		@aboveground_pop -= 1
		puts "One down. \n"
	else
		puts 'There is no one to splode.'
	end
end

Oh no! Those capitalist pigs have missiles! Notice how I put in another if/else statement? This time it tells you if everyone is dead. Now that we made a missile, which subtracts from the aboveground population (missiles can't dig underground), we can play through the scenario. 

def census
	puts "\n Above Ground Population: \n"
	puts @aboveground_pop
	puts "Bunkered Population: \n"
	puts @underground_pop
end

I also made a little thing that shows how many people we have left.

What did we learn?

If you were paying attention, you should've noticed how we made a class, how we made variables accessible throughout the class, and how to preserve your country's population from missiles. The @ symbol is vital to this. All I did was add and subtract, but you can do more! You can make them arrays, and use the various array methods to mess with them. You can just make them strings, or hashes, pretty much any object. Keep researching, this is only a tiny example of what you can do. Good luck comrade!
